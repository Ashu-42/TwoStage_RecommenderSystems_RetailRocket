# Two Stage Wide & Deep Recommender Systems for HomePage using retail Rocket Dataset

## Candidate Generator

### v1: Heuristic-Based Candidate Generator

A heuristic candidate generator was developed using a combination of:

- User historical interactions
- Platform-wide item popularity
- Platform-wide category popularity

The candidate generator aims to predict items that a user is likely to interact with based on their past behavior.

Since this is a heuristic approach, it does not possess the learning capabilities required to generalize well to completely new users. Therefore, candidate generation is currently restricted to **returning users**, i.e., users who have at least one historical session before the current session. This allows the model to leverage past interaction history and user-level behavioral features.

#### Performance

- Candidate Generator Hit Rate (all returning-user queries): **~31%**
- Candidate Generator Hit Rate (queries with at least one future order in the positive set): **~60%+**

## Ranker

### v1: Logistic Regression Ranker

The first ranking model is based on Logistic Regression and is trained using the candidate sets generated by the Candidate Generator.

The objective of the ranker is to estimate the probability of a candidate item being ordered by the user. Candidate items are then ranked in descending order of their predicted probabilities.

Planned feature groups include:

- User interaction history
- Candidate item scores
- Candidate rank
- User-item interaction features
